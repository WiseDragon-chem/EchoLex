EchoLex将逐步支持以下类型的练习：
1. 单词拼写（中文->英文），必须提供首字母辅助，[自动生成+人工选题，概率书写词语变形]
2. 词义选择（英文->中文，选择题） [自动生成+人工选题]
3. 短语、句式记忆（中文->英文语句填空），面向写作 [人工出题]
4. 短文补全（中文->英文），是检测手段 [人工出题]
5. 语境填空（中文->英文语境填空），面向词汇运用 [人工辅助数据库，快速生成]
... ...

# 自动错因分析 - 核心功能
1. 单词拼写题
   1. 识别高频错误点：eg. -ence vs -ance, iou vs iu vs ui vs io, ll vs l, 变形去不去e等
   2. 分析拼写问题还是词义问题：如果拼写和答案比较相似（读音相仿），拼写问题；而读音完全不同，考虑将不同词语的词义搞混了，是词义问题。
2. 句子、短语补全题
   1. 能够分析错误是语法错误还是拼写错误。
   2. 对于拼写错误，需要分析出是单词的变形拼错（变形错误）还是单词的原形拼错（普通/高频拼写）。
   3. 对于语法错误，暂时不予进一步分析。
**该模块需要返回错因**，错因包括：
1. 高频拼写错误
2. 普通拼写错误
3. 拼写词义错误-在拼写模式下引发
4. 辨析词义错误-在词义选择模式下引发
5. 变形错误
6. 语法错误
7. 其他错误

# 主界面
1. 最上方横条存在题单、词库、用户注册/登录三个栏目。
2. 中间偏上(1/4)部位：一个圆角矩形输入框，起到电子词典的功能。
3. 中部(1/4 - 3/4)，一个按钮：echo（回顾）
4. 下部（3/4以下），四个按钮：Schedule、错题、题单题库
[该部分需要随着开发不断修改]
*整体UI风格比较简约，但需动态而富有活力，不过这是后话*

# Echo与Schedule页面 - 关键
schedule用于用户自定义学习规划：例如题目来源，各模块配比、各模块每日/周配额等基础参数
也包括：要点回顾比例，各类型错误选取权重、回顾周期、回顾强度等高级参数
echo会自动混合新知识和旧知识、易错点供给用户。
 - 理想的使用方式是：某用户期望在15天内解决高考3500词，系统可以计算每日配额（需根据错误量动态调节），保证可以在15天后对这些词汇有一个高的正确率。
 - echo也会提供强化突破功能，用户主动选择题目范围、类型、数量等。
进入练习环节后，程序评判+错因分析后保存历史记录。
此外，用户每完成一道题，可以让他做一个评估：
1. “秒懂” (Easy): 大大延长下一次复习的间隔。
2. “想起” (Good): 采用标准的复习间隔。
3. “困难” (Hard): 显著缩短下一次复习的间隔。
4. “忘记” (Again): 在本次学习的稍后阶段立即再次出现。
该步骤能将用户的主观感受量化，为唤醒算法提供数据。当然，也可以让算法根据完成时间估测。

# 词典页面
访问词汇数据库，并返回查询结果。用户可在查询页面一键将该词语加入个人题单（选择掌握水平：会拼写/能认识）

# 题库页面
1. 所有的单词本身就是一个单词拼写题，拥有一个固定永久uid。
2. 词义选择题：若为程序生成，分配临时uid，若干天后销毁；若人类用户保存该题或该题由人类用户编写，分配永久uid
3. 短语、句式、语境填空题分配永久uid。

# 题单页面
处于某些目的，将一些题目归为一类，即为题单。题单仅保存各个题目的uid，临时uid无法被保存。
用户可以设置自己的题单是否公开，在该页面也可以浏览其他用户提供的题单。

# 辅助功能
1. 刻蚀章系统：达成特定里程碑后解锁成就。
2. 积分+访问控制：通过日常任务获取积分，访问非官方的用户题库需要积分解锁，这些积分以一定占比回馈给题单创建者。

# 联网功能
该软件最终需要实现联网：
1. 桌面端：允许离线运行，拥有本地数据库，本地题单、题库数据库。可以择时会随时与服务器同步。
